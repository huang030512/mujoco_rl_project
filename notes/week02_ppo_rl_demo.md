# 加速版第2周任务B：PPO 强化学习训练闭环入门（MuJoCo 单摆控制）

## 1. 任务目标

本任务在已完成的 MuJoCo 单关节模型与 PD 控制实验基础上，进一步引入强化学习训练流程。通过将单关节摆杆模型封装为 Gymnasium 自定义环境，并使用 Stable-Baselines3 中的 PPO（Proximal Policy Optimization）算法进行训练，使智能体能够依据当前关节角度和角速度输出控制力矩，将摆杆从随机初始位置运动至目标角度，并在目标附近稳定保持。

本任务的主要目标如下：

* 建立 MuJoCo 模型与强化学习算法之间的标准环境接口；
* 理解观测空间、动作空间、奖励函数和 episode 终止机制；
* 使用 PPO 完成单关节连续控制任务的训练、保存和加载；
* 通过评估脚本和结果曲线验证控制效果；
* 建立“环境建模—策略训练—模型评估—结果可视化”的最小强化学习闭环。

## 2. 模型与任务设计

### 2.1 单关节摆杆模型

本任务使用 `models/hinge_rod.xml` 作为物理模型文件。模型包含一个绕固定轴旋转的单自由度摆杆，并配置一个电机执行器输出控制力矩。

在初始版本中，杆体几何中心与旋转关节位置接近重合，导致杆体转动后重力对关节产生的转动力矩不明显。在该模型下，即使控制力矩趋近于零，杆体也可能保持在目标角附近，因此控制任务的物理意义不足。

为构造更合理的控制任务，后续将杆体修改为关节位于杆体一端、质心远离旋转轴的单摆结构。修改后，摆杆抬起至目标角度时需要持续输出一定控制力矩抵抗重力，因此可以更真实地验证强化学习策略的闭环控制能力。

修改后的模型具有以下特点：

* 关节类型：`hinge`；
* 控制对象：单自由度摆杆；
* 执行器类型：`motor`；
* 电机控制范围：`[-5, 5]`；
* 杆体质心与关节分离，重力能够对关节产生实际转动力矩；
* 关节配置少量阻尼，以降低不必要的高频振荡。

### 2.2 控制目标

摆杆目标角度设为：

```python
target_angle = np.pi / 4
```

即：

```text
45°
```

强化学习策略需要根据当前状态输出控制动作，使摆杆从随机初始角度运动至 `45°` 附近，并在存在重力影响的情况下保持稳定。

## 3. Gymnasium 环境封装

### 3.1 环境文件

自定义强化学习环境实现于：

```text
src/hinge_rod_env.py
```

该环境继承自 `gymnasium.Env`，并实现以下核心接口：

* `reset()`：重置 MuJoCo 仿真状态并返回初始观测；
* `step(action)`：接收策略动作、推进仿真、计算奖励并返回下一状态；
* `action_space`：定义策略允许输出的动作范围；
* `observation_space`：定义神经网络能够接收的状态输入范围。

### 3.2 观测空间

环境观测量定义为：

```python
obs = [theta, theta_dot]
```

其中：

* `theta`：当前关节角度，单位为 `rad`；
* `theta_dot`：当前关节角速度，单位为 `rad/s`。

观测空间写为：

```python
self.observation_space = spaces.Box(
    low=np.array([-np.inf, -np.inf], dtype=np.float32),
    high=np.array([np.inf, np.inf], dtype=np.float32),
    dtype=np.float32
)
```

该设计使策略网络能够同时获知当前位置和当前运动趋势，从而输出具备制动和稳定保持能力的控制动作。

### 3.3 动作空间与力矩映射

PPO 输出的动作被归一化为：

```python
action ∈ [-1, 1]
```

对应环境定义为：

```python
self.action_space = spaces.Box(
    low=np.array([-1.0], dtype=np.float32),
    high=np.array([1.0], dtype=np.float32),
    dtype=np.float32
)
```

环境内部再将归一化动作映射为实际电机力矩：

```python
torque = action * max_torque
```

其中：

```python
max_torque = 5.0
```

因此实际施加到 MuJoCo 电机执行器上的控制力矩范围为：

```text
[-5, 5]
```

采用动作归一化的原因是：策略网络只需学习统一范围内的控制输出，而具体执行器量纲和最大输入范围由环境内部完成映射。这种设计更适合后续扩展至其他机器人执行器控制任务。

### 3.4 初始状态随机化

在每个 episode 开始时，摆杆从一定范围内的随机初始角度出发：

```python
self.data.qpos[0] = self.np_random.uniform(
    low=-np.pi / 6,
    high=np.pi / 6
)
```

即初始角度位于：

```text
[-30°, 30°]
```

关节初始角速度设为：

```python
self.data.qvel[0] = 0.0
```

采用随机初始角度可以避免策略只记住单一初始状态下的动作序列，使其能够学习根据当前观测进行反馈控制。

### 3.5 控制周期与 episode 长度

MuJoCo 模型的基础物理仿真步长为：

```text
0.002 s
```

环境中设置：

```python
frame_skip = 5
```

即 PPO 每输出一个动作，该动作在 MuJoCo 中连续作用 5 个物理仿真步。因此强化学习控制周期为：

```text
0.002 s × 5 = 0.01 s
```

对应控制频率为：

```text
100 Hz
```

每个 episode 最多执行：

```python
max_episode_steps = 500
```

因此单个 episode 总仿真时间为：

```text
500 × 0.01 s = 5 s
```

该设计能够让策略在完整时间段内持续优化控制效果，而不是仅在某一瞬间经过目标角度后立即结束回合。

## 4. 奖励函数设计

### 4.1 奖励函数形式

本任务采用由角度误差、角速度和控制力矩共同构成的奖励函数：

```python
reward = -(
    5.0 * angle_error ** 2
    + 0.01 * theta_dot ** 2
    + 0.001 * torque ** 2
)
```

其中：

* `angle_error`：当前角度与目标角度之间的误差；
* `theta_dot`：当前关节角速度；
* `torque`：当前策略输出对应的实际控制力矩。

为防止关节旋转多圈后角度误差不合理增大，误差通过下式限制在 `[-π, π]` 范围内：

```python
angle_error = np.arctan2(
    np.sin(theta - target_angle),
    np.cos(theta - target_angle)
)
```

### 4.2 各奖励项作用

| 奖励项                      | 作用                     |
| ------------------------ | ---------------------- |
| `5.0 * angle_error ** 2` | 强化摆杆接近目标角度的需求，提高最终角度精度 |
| `0.01 * theta_dot ** 2`  | 抑制摆杆高速摆动，鼓励到达目标后稳定停止   |
| `0.001 * torque ** 2`    | 限制过大的控制输入，避免策略长期依赖饱和力矩 |

在初始训练中，角度误差惩罚权重较小，模型虽然能够到达目标附近并停稳，但存在一定稳态偏差。随后提高角度误差惩罚权重并延长训练步数后，最终角度误差进一步降低。

## 5. PPO 训练配置

### 5.1 训练脚本

PPO 训练脚本实现于：

```text
src/train_ppo_hinge_rod_full.py
```

训练使用 Stable-Baselines3 提供的 PPO 算法，策略网络类型为多层感知机策略：

```python
PPO("MlpPolicy", env, ...)
```

### 5.2 主要训练参数

| 参数                |         设置值 | 说明                      |
| ----------------- | ----------: | ----------------------- |
| 算法                |         PPO | 连续动作控制策略优化算法            |
| 策略网络              | `MlpPolicy` | 输入低维状态观测的全连接神经网络        |
| `learning_rate`   |      `3e-4` | 策略网络学习率                 |
| `batch_size`      |        `64` | 每次网络更新使用的样本数量           |
| `n_steps`         |       `256` | 每次策略更新前采集的 rollout 长度   |
| `total_timesteps` |    `100000` | 总训练交互步数                 |
| `device`          |       `cpu` | PPO 与低维 MLP 策略使用 CPU 训练 |

由于本任务的观测空间和动作空间维度较低，策略网络规模较小。PPO 在此类 `MlpPolicy` 任务上使用 GPU 并不能明显提升训练速度，因此最终将训练设备设置为 CPU。

## 6. 训练过程与问题修正

### 6.1 初始环境存在的问题

初始环境中，episode 的终止条件设置为：

```python
terminated = abs(theta - target_angle) < 0.01
```

该条件意味着，只要摆杆在某一瞬间经过目标角度，回合就会被视为完成。这会导致策略可能学习到“快速扫过目标角度”的行为，而不是稳定保持目标姿态。

此外，初始版本中每一个策略动作仅执行一个 MuJoCo 物理步，控制周期为 `0.002 s`，策略输出变化过于频繁，不利于入门阶段分析控制过程。

### 6.2 环境修正

为提高任务合理性，环境作出以下调整：

* 去除“瞬间经过目标即结束 episode”的逻辑；
* 将每个 episode 固定为最多 `500` 个强化学习控制步；
* 设置 `frame_skip = 5`，使控制周期调整为 `0.01 s`；
* 在奖励函数中同时引入角度误差、角速度和控制力矩惩罚；
* 引入随机初始角度，提高策略对不同初始状态的适应性。

### 6.3 模型结构修正

在初始摆杆结构中，杆体质心与旋转关节位置接近重合，导致稳态保持时几乎不需要持续控制力矩。虽然角度曲线能够收敛，但任务无法充分体现策略在重力作用下的控制能力。

因此，将摆杆模型修改为关节位于杆体一端、杆体向下延伸的单摆结构。修正后，摆杆抬起至 `45°` 附近时需要持续输出非零力矩抵抗重力，控制任务更符合真实关节控制场景。

## 7. 模型评估

### 7.1 评估脚本

训练完成后，使用以下脚本加载已保存模型并进行确定性策略评估：

```text
src/eval_ppo_hinge_rod.py
```

评估阶段使用：

```python
action, _ = model.predict(obs, deterministic=True)
```

以排除训练过程中随机探索对结果的影响，重点检验最终策略已经学到的稳定控制能力。

### 7.2 成功判据

环境中定义稳定成功状态为：

```python
abs(angle_error) < np.deg2rad(3.0)
and abs(theta_dot) < np.deg2rad(5.0)
```

即：

* 角度误差小于 `3°`；
* 角速度绝对值小于 `5°/s`。

该判据能够区分“高速经过目标角度”和“真正稳定保持在目标附近”两种情况。

### 7.3 最终结果

在修改后的单摆模型上重新训练和评估后，得到如下结果：

| 指标     |         结果 |
| ------ | ---------: |
| 目标角度   |   `45.00°` |
| 最终角度   |   `45.75°` |
| 最终角度误差 |    `0.75°` |
| 最终状态   | 稳定保持在目标角附近 |
| 稳态控制力矩 |   约为 `1.0` |

最终评估结果表明，PPO 策略能够将摆杆从初始位置快速抬起至目标角度附近，并在重力作用下通过持续输出控制力矩保持稳定。

## 10. 学习总结

通过本任务，完成了从经典控制 demo 向强化学习控制闭环的过渡。相较于此前直接编写 PD 控制律的方式，PPO 不需要显式设计比例项和微分项，而是通过与环境交互，根据奖励函数逐步学习控制策略。

本任务中重点理解并完成了以下内容：

* MuJoCo 模型如何封装为 Gymnasium 强化学习环境；
* `reset()` 与 `step()` 在训练闭环中的作用；
* 观测空间、动作空间与实际电机力矩之间的对应关系；
* 随机初始状态对策略泛化能力的意义；
* 奖励函数如何同时约束角度误差、运动速度和控制输入；
* episode 时间限制与 `frame_skip` 对训练任务定义的影响；
* 环境物理建模合理性对强化学习结果解释的重要影响；
* PPO 模型的训练、保存、加载、定量评估和结果可视化流程。

最终结果表明，训练得到的 PPO 策略能够在具有重力影响的单摆模型上，将关节稳定控制至 `45°` 目标角附近，最终误差约为 `0.75°`，并在稳态阶段持续输出非零力矩抵抗重力。至此，完成了一个可复现、可评估、可展示的 MuJoCo 单关节 PPO 强化学习控制闭环实验。
