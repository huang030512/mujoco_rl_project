## 2. 仿真对象与模型设置

本次实验使用的模型文件为 `models/simple_box.xml`。模型中包含一个地面平面和一个可自由运动的盒子。盒子通过 `<freejoint/>` 与世界坐标系连接，因此能够在三维空间中进行平移和旋转。

模型中的主要设置如下：

| 项目 | 设置 |
| --- | --- |
| 模型文件 | `models/simple_box.xml` |
| 仿真对象 | 自由下落盒子 |
| 关节类型 | `<freejoint/>` |
| 初始高度 | 约 0.5 m |
| 盒子半高 | 0.1 m |
| 重力加速度 | `0 0 -9.81` m/s² |
| 总仿真时间 | 2.0 s |
| 时间步长 | `model.opt.timestep` |

盒子的初始位置设置在地面上方。在重力作用下，盒子首先发生竖直向下运动；当盒子底面与地面发生接触后，MuJoCo 的碰撞与接触求解机制会产生支撑和接触响应，使盒子不再继续无限下落。

需要注意的是，程序记录状态的操作发生在第一次调用 `mujoco.mj_step(model, data)` 之后，因此绘图数据中的第一个高度值为 `0.49996076 m`，略小于模型中设置的初始高度 `0.5 m`。这说明在记录第一个数据点之前，盒子已经经历了一个仿真时间步长的下落过程。

## 3. 仿真循环流程

本次仿真结果分析使用脚本 `src/simulate_box_plot.py`。脚本首先加载 XML 模型，并基于模型创建用于保存动态状态的 `MjData` 对象：

```python
model = mujoco.MjModel.from_xml_path("models/simple_box.xml")
data = mujoco.MjData(model)
```

随后，程序设置总仿真时间为 `2.0 s`，并通过模型中的时间步长计算仿真循环总步数：

```python
sim_time = 2.0
dt = model.opt.timestep
steps = int(sim_time / dt)
```

在仿真开始前，程序分别建立三个列表，用于保存每个时间步对应的时间、竖直位置和竖直速度：

```python
times = []
z_positions = []
vz_values = []
```

核心仿真循环如下：

```python
for i in range(steps):
    mujoco.mj_step(model, data)
    times.append(data.time)
    z_positions.append(data.qpos[2])
    vz_values.append(data.qvel[2])
```

在每一次循环中，程序首先调用：

```python
mujoco.mj_step(model, data)
```

该函数会使 MuJoCo 仿真向前推进一个时间步长 `dt`。在这一过程中，仿真器会根据当前模型参数、重力、物体速度以及接触状态更新系统的时间、位置和速度等状态变量。

每调用一次 `mujoco.mj_step(model, data)`，仿真时间 `data.time` 会增加一个 `dt`。由于盒子受到竖直向下的重力作用，其竖直位置 `data.qpos[2]` 会在落地前持续减小，竖直速度 `data.qvel[2]` 会由接近 0 逐渐变为负值。当盒子接触地面后，接触力会阻止盒子继续穿过地面，并使速度发生快速变化。

循环结束后，程序将列表转换为 NumPy 数组，用于后续曲线绘制：

```python
times = np.array(times)
z_positions = np.array(z_positions)
vz_values = np.array(vz_values)
```

在绘图部分，程序分别生成盒子高度随时间变化的曲线图和竖直速度随时间变化的曲线图，并将图片保存至 `assets/` 文件夹中：

```text
assets/week01_box_free_fall_position.png
assets/week01_box_free_fall_velocity.png
```

## 4. 状态变量说明

### 4.1 自由关节的状态维度

本实验中的盒子使用了 `<freejoint/>`。自由关节允许刚体在三维空间中进行三个方向的平移和三个方向的旋转，因此模型状态中包含位置与姿态信息。

通过模型结构检查可知：

```text
nq = 7
nv = 6
```

其中，`nq` 表示广义位置变量的维度，`nv` 表示广义速度变量的维度。

### 4.2 qpos 的含义

对于自由关节，`qpos` 共有 7 个分量：

| 分量 | 含义 |
| --- | --- |
| `qpos[0]` | 盒子在 x 方向的位置 |
| `qpos[1]` | 盒子在 y 方向的位置 |
| `qpos[2]` | 盒子在 z 方向的位置，即竖直高度 |
| `qpos[3:7]` | 表示盒子姿态的四元数 |

本实验主要关注竖直方向的自由落体过程，因此使用：

```python
data.qpos[2]
```

作为盒子竖直高度的记录变量，并将其保存到 `z_positions` 中。

`qpos` 的后三个维度并不是欧拉角，而是四个分量组成的四元数。四元数用于表示三维姿态，相比欧拉角能够避免部分姿态表达中的奇异问题。本次实验中盒子主要进行竖直下落，暂不需要对四元数进行进一步计算，但需要明确其作用是表示刚体姿态。

### 4.3 qvel 的含义

对于自由关节，`qvel` 共有 6 个分量：

| 分量 | 含义 |
| --- | --- |
| `qvel[0]` | x 方向线速度 |
| `qvel[1]` | y 方向线速度 |
| `qvel[2]` | z 方向线速度，即竖直速度 |
| `qvel[3]` | 绕 x 轴的角速度 |
| `qvel[4]` | 绕 y 轴的角速度 |
| `qvel[5]` | 绕 z 轴的角速度 |

本实验使用：

```python
data.qvel[2]
```

作为盒子的竖直速度，并将其保存到 `vz_values` 中。

在盒子下落阶段，由于 z 轴正方向向上，而重力方向向下，因此 `qvel[2]` 会逐渐变为负值。接触地面后，接触响应会使其数值快速发生变化，并最终趋近于 0。

## 5. 仿真结果分析

### 5.1 高度变化曲线分析

盒子竖直高度随时间变化的曲线保存在：

```text
assets/week01_box_free_fall_position.png
```

根据实际运行结果，程序记录到的初始高度为：

```text
0.49996076 m
```

该值略小于模型设定的 `0.5 m`，原因是程序在第一次执行 `mujoco.mj_step(model, data)` 后才保存第一个状态数据，因此盒子已经在重力作用下发生了一个时间步长的下落。

在仿真开始后，盒子高度随时间持续下降，说明盒子在重力作用下进行自由落体运动。大约在：

```text
0.25 s
```

时，盒子首次接触地面。

接触地面后，盒子高度没有继续减小，而是在短暂变化后稳定在：

```text
0.09989224457978375 m
```

附近。

该稳定高度约为 `0.1 m`，与盒子几何模型的半高一致。由于地面平面位于 `z = 0` 附近，当盒子底面与地面接触时，盒子质心位置应位于约 `z = 0.1 m` 处。因此，稳定高度接近 `0.1 m` 是符合模型结构和物理直觉的结果。

### 5.2 竖直速度曲线分析

盒子竖直速度随时间变化的曲线保存在：

```text
assets/week01_box_free_fall_velocity.png
```

在仿真开始阶段，盒子由接近静止状态开始下落。在重力作用下，其竖直速度 `v_z` 逐渐变为负值，说明盒子沿 z 轴负方向运动，即向地面下落。

在盒子接触地面之前，速度负值的绝对值不断增大，反映了自由落体过程中盒子持续加速下落的状态。

当盒子约在 `0.25 s` 接触地面后，竖直速度曲线发生明显变化。由于地面向盒子提供接触响应，盒子原有的向下速度被迅速削弱；同时，从实际曲线观察到盒子在落地后出现了短暂上跳现象，因此接触后的短时间内竖直速度可能出现由负值向接近 0 或短暂正值变化的过程。

随后，随着接触响应逐渐稳定，盒子的竖直速度最终趋近于 0，表明盒子已经稳定停留在地面上。

### 5.3 接触地面后的现象分析

本次仿真中，盒子落地后出现了短暂上跳，而不是立即完全静止。这种现象说明在接触发生的瞬间，盒子具有一定的向下速度，接触求解过程中产生的支撑响应会使盒子的运动状态迅速改变，从而表现为高度曲线的短暂回升以及速度曲线的快速变化。

该现象符合刚体碰撞仿真的基本直觉：盒子在空中受到重力作用加速下落；接触地面时，地面提供阻止穿透的接触力；由于接触过程不是无限刚性且具有动态响应，盒子在最终稳定前可能出现短暂弹跳或振荡。

总体来看，本次仿真结果符合预期：

1. 下落前期，盒子高度持续下降，竖直速度持续变为负值；
2. 接触地面时，高度曲线和速度曲线均发生明显变化；
3. 落地后，盒子出现短暂上跳；
4. 最终高度稳定在约 `0.1 m`，竖直速度趋近于 0；
5. 盒子未继续穿过地面，说明模型中的接触碰撞机制正常发挥作用。

### 6.2 对关键概念的理解

通过本次实验，可以得到以下理解：

1. `mujoco.mj_step(model, data)` 的作用是将仿真系统向前推进一个时间步长，并更新当前时刻下刚体的位置、速度、接触状态等数据。
2. 对于本实验，每执行一次 `mujoco.mj_step(model, data)`，`data.time` 增加一个 `model.opt.timestep`。
3. `qpos[2]` 表示盒子在世界坐标系 z 方向的位置，因此可用于描述自由落体过程中的高度变化。
4. `qvel[2]` 表示盒子在 z 方向的线速度。下落时其值为负，落地后由于接触响应而发生快速变化，并最终趋近于 0。
5. 自由关节的 `qpos` 为 7 维，是因为其同时保存了三个位置分量和四元数姿态；`qvel` 为 6 维，是因为刚体速度由三个线速度分量和三个角速度分量组成。
6. 地面与盒子发生接触后，MuJoCo 会根据接触约束和碰撞响应计算支撑作用，因此盒子不会继续无限下落。

### 6.3 当前仍需进一步确认的问题

本次实验已经能够观察到自由落体和落地响应的基本规律，但后续仍可以进一步完善以下内容：

1. 通过程序自动判断首次接触地面的准确时间，而不仅依靠曲线进行近似观察；
2. 进一步观察竖直速度在落地瞬间的最小值和回弹阶段的最大值，以更准确地描述碰撞响应；
3. 了解 MuJoCo 中与接触刚度、阻尼和回弹特性相关的参数对曲线变化的影响。
