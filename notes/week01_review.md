## 2. 项目文件与作用说明

### 2.1 `models/simple_box.xml`

该文件用于定义第一周自由落体实验的 MuJoCo 模型和仿真场景，包括地面、盒子本体、初始位置、质量、几何尺寸以及自由关节等信息。

我对该文件作用的理解是：

XML 文件中通过 `<geom>` 定义了地面作为盒子落下的支撑，通过 `<body>` 和 `<geom>` 定义盒子的尺寸、质量和初始位置，`<freejoint/>` 允许盒子在空间中自由下落。MuJoCo 根据这些定义生成静态模型 `MjModel`，并在仿真中使用 `MjData` 更新状态。

### 2.2 `src/inspect_model.py`

该文件用于加载 `simple_box.xml`，并读取模型中的结构信息，例如 body、joint、geom、actuator 以及相关参数。

我实际观察到该脚本打印的信息包括：

* body 列表：`box`、`floor` 等；
* joint 列表：自由关节信息；
* geom 列表：盒子尺寸和地面大小；
* actuator 信息为空，本实验未定义驱动器。

我对该脚本作用的理解是：

该脚本用于检查模型是否按照 XML 文件中的配置正确加载，确认 body、joint、geom 的属性和数量，为后续仿真运行和模型修改做好准备。

### 2.3 仿真循环与绘图脚本

我实际用于自由落体仿真的脚本为：

* 仿真脚本：`src/run_simple_box.py`
* 绘图脚本：`src/simulate_box_plot.py`

在脚本中，`mj_step(model, data)` 的作用是：

按照模型参数推进仿真一个时间步，并更新 `MjData` 中的状态，例如位置 `qpos`、速度 `qvel` 和时间 `time` 等。

在脚本中，`qpos[2]` 与 `qvel[2]` 的作用分别是：

`qpos[2]` 记录盒子在 z 轴方向的位置，可用于观察盒子的高度变化；`qvel[2]` 记录盒子在 z 轴方向的速度，可用于分析盒子下落、触地和反弹过程。

### 2.4 第一周完整技术链路

我对第一周技术链路的理解如下：

```text
models/simple_box.xml
        ↓
MuJoCo 加载 XML 模型
        ↓
MjModel 保存模型结构和固定参数
MjData 保存运行过程中的动态状态
        ↓
mj_step(model, data) 推进仿真
        ↓
读取 qpos[2] 和 qvel[2]
        ↓
绘制盒子竖直位置与竖直速度曲线
        ↓
分析自由落体、触地、短暂反弹和最终稳定现象
```

我用自己的话解释这条链路：

MuJoCo 首先根据 XML 文件创建 `MjModel`，然后使用 `MjData` 保存仿真运行过程中的状态。程序通过循环调用 `mj_step()` 推进仿真，并持续读取盒子的竖直位置和竖直速度。最后，将记录的数据绘制成曲线，从而观察盒子从初始高度自由下落、接触地面、发生短暂反弹并最终稳定的完整过程。

## 3. 已掌握的核心概念

### 3.1 `MjModel` 与 `MjData`

`MjModel` 用于保存 MuJoCo 仿真中的固定模型参数，例如 body、joint、geom 的结构信息，以及质量、重力和仿真步长等参数。

`MjData` 用于保存仿真运行中不断变化的状态，例如当前时间、位置 `qpos` 和速度 `qvel` 等。

我自己的理解是：

`MjModel` 保存的是模型本身固定不变的结构和参数，`MjData` 保存的是模型在运行过程中不断变化的状态。在仿真循环中，`MjModel` 通常保持不变，而 `MjData` 会随着每次仿真推进不断更新。

### 3.2 自由关节与 `nq`、`nv`

在本实验中，盒子使用 `<freejoint/>`，因此可以在三维空间中自由平移和旋转。

* `nq = 7`：包含 3 个平移量和 4 个四元数旋转量。
* `nv = 6`：包含 3 个线速度和 3 个角速度。

我自己的理解是：

自由关节使盒子能够在空间中自由运动，因此需要使用 3 个位置量表示盒子的空间位置，并使用 4 个四元数分量表示盒子的旋转姿态。速度方面，则分别使用 3 个线速度分量和 3 个角速度分量描述运动状态。

### 3.3 `qpos[2]` 与 `qvel[2]`

在当前自由落体实验中：

* `qpos[2]` 表示盒子在 z 轴方向的位置，可用于观察高度变化。
* `qvel[2]` 表示盒子在 z 轴方向的速度，负值说明盒子正在向下运动。

我自己的理解是：

通过 `qpos[2]` 可以观察盒子的高度是否持续下降、是否触地并最终稳定；通过 `qvel[2]` 可以观察盒子的下落速度变化，以及触地后是否出现速度方向改变和短暂反弹现象。

### 3.4 `mj_step()`

`mj_step(model, data)` 用于按照当前模型和物理参数，将仿真向前推进一个时间步，并更新 `MjData` 中的运行状态。

我自己的理解是：

每调用一次 `mj_step()`，仿真时间都会向前推进，同时 `qpos`、`qvel` 和 `time` 等状态数据也会发生更新，因此连续调用该函数可以得到盒子整个自由落体过程中的运动变化。

## 4. 自由落体实验结果回顾

### 4.1 实际实验结果

根据我实际生成的曲线和运行结果：

* 初始高度：约 `0.5 m`
* 第一次触地时间：约 `0.25 s`
* 最终稳定高度：约 `0.1 m`
* 是否观察到短暂反弹：是

对应结果图片或视频路径：

* 位置曲线：`assets/week01_box_free_fall_position.png`
* 速度曲线：`assets/week01_box_free_fall_velocity.png`
* 录屏或演示文件：`assets/simple_box_viewer.png`

### 4.2 实验现象分析

我观察到盒子首先在重力作用下向下运动，其竖直位置逐渐降低，竖直速度逐渐变为负值。盒子接触地面时，由于仍具有向下运动的速度，地面接触作用会阻止盒子继续穿透，因此盒子会出现轻微反弹或振荡现象。随后，盒子的竖直速度逐渐趋近于零，高度也稳定在地面支撑位置附近。

在曲线中，该现象表现为盒子的高度先持续降低，触地后出现小幅波动，最终保持稳定；速度曲线则表现为速度先沿负方向变化，在触地后短暂发生反向变化，并最终趋近于零。

## 5. 本周遇到的问题与解决过程

### 问题 1：模型属性读取方式错误

遇到的问题：

在编写 `src/inspect_model.py` 时，初期尝试直接访问 `model.body_names`、`model.joint_names` 等属性，程序出现 `AttributeError`，说明当前 MuJoCo Python 接口中不存在这些直接可访问的名称列表属性。

解决过程：

通过重新查看模型读取方式，改为按照模型中 body、joint 和 geom 的数量进行遍历，并使用正确的接口读取各对象名称和参数，最终成功打印模型结构信息。

### 问题 2：模型元素或仿真现象理解不清

遇到的问题：

初期不清楚 `<freejoint/>` 与 `nq=7`、`nv=6` 之间的关系，也不能准确解释 `qpos[2]`、`qvel[2]` 和自由落体实验现象之间的对应关系。

解决过程：

通过重新阅读 XML 模型文件、查看仿真脚本输出以及分析位置和速度曲线，我逐步理解了自由关节允许盒子进行空间平移和旋转，`qpos[2]` 表示竖直位置，`qvel[2]` 表示竖直速度，并能够根据曲线解释盒子自由下落、触地和短暂反弹的过程。

### 问题 3：模型文件内容误删导致盒子消失

遇到的问题：

在修改 `simple_box.xml` 的过程中，曾误将 `<freejoint/>` 和定义盒子几何体的 `<geom>` 内容一起删除，导致运行可视化程序后看不到原本的蓝色盒子。

解决过程：

重新检查 XML 文件内容后，恢复盒子 body 内部的 `<freejoint/>` 和盒子几何体 `<geom>` 定义，使模型重新能够正确显示并进行自由落体仿真。

## 7. 第2周准备事项：XML 元素理解

### 7.1 当前能够识别的 XML 元素

* **模型名称**

  ```xml
  <mujoco model="simple_box">
  ```

  该标签表示当前 MuJoCo 模型的名称为 `simple_box`，整个模型文件中的仿真配置均包含在该根节点内部。

* **角度单位设置**

  ```xml
  <compiler angle="degree"/>
  ```

  该标签表示模型文件中使用角度参数时，角度单位采用度。

* **仿真参数**

  ```xml
  <option timestep="0.002" gravity="0 0 -9.81"/>
  ```

  其中，`timestep="0.002"` 表示仿真每次推进的时间步长为 `0.002 s`；`gravity="0 0 -9.81"` 表示重力沿 z 轴负方向作用。

* **地面**

  ```xml
  <geom name="floor"
        type="plane"
        size="2 2 0.1"
        rgba="0.8 0.8 0.8 1"/>
  ```

  地面采用 `plane` 类型的几何体表示，用于与下落的盒子发生接触，并最终支撑盒子稳定停留。

* **盒子本体**

  ```xml
  <body name="box" pos="0 0 0.5">
  ```

  该标签定义名为 `box` 的刚体，其初始位置位于 z 轴高度 `0.5 m` 处。

* **盒子几何体**

  ```xml
  <geom name="box_geom"
        type="box"
        size="0.1 0.1 0.1"
        mass="1"
        rgba="0.2 0.5 0.8 1"/>
  ```

  该标签定义盒子的几何形状、尺寸、质量和显示颜色。其中，`type="box"` 表示几何体类型为长方体，`mass="1"` 表示盒子质量为 `1 kg`。

* **自由关节**

  ```xml
  <freejoint/>
  ```

  该标签使盒子能够在三维空间中自由平移和旋转，因此盒子可以在重力作用下发生自由下落运动。
