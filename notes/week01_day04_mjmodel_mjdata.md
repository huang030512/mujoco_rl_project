## 2. MjModel 与 MjData 的区别

### 2.1 MjModel

`MjModel` 用于保存从 XML 模型文件加载得到的模型结构和固定参数，例如：

* 模型中的刚体 `body`；
* 关节 `joint`；
* 几何体 `geom`；
* 执行器 `actuator`；
* 重力参数；
* 仿真时间步长；
* 模型的位置、速度和控制维度。

在当前 `simple_box.xml` 模型中，`MjModel` 保存了地面、方块、自由关节、重力和时间步长等信息。

### 2.2 MjData

`MjData` 用于保存仿真运行过程中不断变化的动态状态，例如：

* 当前仿真时间 `data.time`；
* 当前广义位置 `data.qpos`；
* 当前广义速度 `data.qvel`；
* 当前控制输入 `data.ctrl`。

### 2.3 两者关系

`MjModel` 描述模型的结构和固定物理参数，回答“模型是什么、如何运动”的问题；`MjData` 保存仿真过程中的当前状态，回答“模型现在运动到了什么状态”的问题。

调用：

```python
mujoco.mj_step(model, data)
```

时，MuJoCo 会根据 `model` 中保存的物理参数和模型结构，将 `data` 中的状态向前推进一个仿真时间步。

当前模型中的时间步长为：

```text
0.002 s
```

因此，每调用一次 `mj_step()`，仿真时间推进 `0.002 s`。

## 4. nq、nv、nu、qpos、qvel 与 ctrl 的含义

| 字段     | 当前输出                      | 含义                | 当前模型中的解释                                        |
| ------ | ------------------------- | ----------------- | ----------------------------------------------- |
| `nq`   | `7`                       | 广义位置向量 `qpos` 的长度 | 当前方块具有自由关节，位置由三维平移位置和四元数姿态组成，因此为 `3 + 4 = 7`    |
| `nv`   | `6`                       | 广义速度向量 `qvel` 的长度 | 自由刚体具有三个方向的线速度和三个方向的角速度，因此为 `3 + 3 = 6`         |
| `nu`   | `0`                       | 控制输入向量 `ctrl` 的长度 | 当前模型没有定义 actuator，因此没有控制输入通道                    |
| `qpos` | `[0. 0. 0.5 1. 0. 0. 0.]` | 当前广义位置            | 方块初始位置为 `(0, 0, 0.5)`，初始姿态为单位四元数 `(1, 0, 0, 0)` |
| `qvel` | `[0. 0. 0. 0. 0. 0.]`     | 当前广义速度            | 初始时刻方块没有线速度和角速度                                 |
| `ctrl` | `[]`                      | 执行器控制输入           | 当前模型没有 actuator，因此控制输入数组为空                      |

### 4.1 qpos 的组成

当前模型中的方块通过 `<freejoint/>` 与世界连接。对于自由关节刚体：

```text
qpos = [x, y, z, qw, qx, qy, qz]
```

当前输出：

```text
[0.  0.  0.5 1.  0.  0.  0. ]
```

表示：

* 方块初始位置为 `x = 0 m`、`y = 0 m`、`z = 0.5 m`；
* 方块初始姿态四元数为 `(1, 0, 0, 0)`，表示初始状态没有旋转。

### 4.2 qvel 的组成

自由刚体的速度状态为：

```text
qvel = [vx, vy, vz, wx, wy, wz]
```

其中：

* `vx`、`vy`、`vz` 为三个方向的线速度；
* `wx`、`wy`、`wz` 为三个方向的角速度。

当前输出全部为 `0`，说明仿真初始化时方块处于静止状态。

### 4.3 ctrl 的作用

`ctrl` 用于向模型中的 actuator 提供主动控制输入，例如力、力矩或关节控制目标。

当前模型没有定义 actuator，因此：

```text
nu = 0
ctrl = []
```

这说明当前模型只能依靠重力和接触碰撞等物理作用发生运动，暂时不能通过 `ctrl` 主动控制方块。

## 5. body、joint、geom 与 actuator 的关系

| 结构         | 作用                   | 当前模型中的对象           |
| ---------- | -------------------- | ------------------ |
| `body`     | 表示具有质量、惯量和位姿的刚体或场景节点 | `world`、`box`      |
| `joint`    | 定义 body 相对父节点可以如何运动  | 方块上的自由关节           |
| `geom`     | 定义显示和碰撞使用的几何形状       | `floor`、`box_geom` |
| `actuator` | 为关节或运动自由度提供主动控制输入    | 当前模型中没有            |

当前模型结构可以表示为：

```text
world
├── floor
└── box
    ├── freejoint
    └── box_geom
```

其中：

* `world` 是整个仿真场景的根节点；
* `floor` 是地面几何体；
* `box` 是参与运动的方块刚体；
* `freejoint` 使方块能够在三维空间自由平移和旋转；
* `box_geom` 定义方块的碰撞形状和显示形状。

## 6. Body 检查结果分析

### 6.1 输出结果

```text
Body 0: name=world, mass=0.0, inertia=[0. 0. 0.]
Body 1: name=box, mass=1.0, inertia=[0.00666667 0.00666667 0.00666667]
```

### 6.2 分析

当前模型包含两个 body：

* `world` 为仿真场景根节点，不是参与运动的实体，因此其质量和转动惯量均为 `0`；
* `box` 为实际参与自由落体运动的刚体，其质量为 `1.0 kg`。

`box` 的转动惯量为：

```text
[0.00666667 0.00666667 0.00666667]
```

三个方向的转动惯量相同，说明该方块在三个方向上的几何尺寸一致，具有对称结构。

当前方块的实际边长为 `0.2 m`，质量为 `1.0 kg`。均匀正方体绕任一中心主轴的理论转动惯量为：

```text
I = (1 / 12) × m × (0.2² + 0.2²)
  = 0.00666667 kg·m²
```

该理论结果与 MuJoCo 输出结果一致。

---

## 7. Joint 检查结果分析

### 7.1 输出结果

```text
Joint 0: name=None, range=[0. 0.]
```

### 7.2 分析

当前模型中包含一个自由关节，对应 XML 文件中的：

```xml
<freejoint/>
```

关节名称显示为 `None`，是因为 XML 文件中没有显式设置该关节的 `name` 属性。

自由关节允许方块在三维空间中进行：

* 沿 `x`、`y`、`z` 三个方向的平移；
* 绕三个方向的旋转。

输出中的：

```text
range=[0. 0.]
```

不表示方块无法运动。自由关节不采用普通转动关节或滑动关节那样的上下限范围进行约束，因此该输出不能理解为方块的运动范围被限制为零。

## 8. Geom 检查结果分析

### 8.1 输出结果

```text
Geom 0: name=floor, size=[2.  2.  0.1]
Geom 1: name=box_geom, size=[0.1 0.1 0.1]
```

### 8.2 分析

当前模型包含两个 geom：

| Geom 名称    | 作用            | `size` 输出       |
| ---------- | ------------- | --------------- |
| `floor`    | 地面几何体，用于碰撞和显示 | `[2. 2. 0.1]`   |
| `box_geom` | 方块几何体，用于碰撞和显示 | `[0.1 0.1 0.1]` |

对于 `box` 类型的 geom，MuJoCo 中的 `size` 表示三个方向的半边长。因此：

```text
box_geom 的实际尺寸 = 0.2 m × 0.2 m × 0.2 m
```

这也解释了自由落体实验中，方块落在地面上后，其中心高度最终稳定在：

```text
z ≈ 0.100 m
```

因为方块中心距离下表面的距离正好为 `0.1 m`。

### 9.1 输出结果


该标题下没有任何 actuator 输出。

### 9.2 分析


## 10. 额外练习：自由落体状态观察

### 10.1 状态输出

通过运行 `src/simulate_box.py`，得到以下状态输出：

```text
t=0.002s, x=0.000m, y=0.000m, z=0.500m, vz=-0.020m/s
t=0.202s, x=0.000m, y=0.000m, z=0.298m, vz=-1.982m/s
t=0.402s, x=-0.000m, y=-0.000m, z=0.099m, vz=0.035m/s
t=0.602s, x=0.000m, y=-0.000m, z=0.100m, vz=0.000m/s
t=0.802s, x=0.000m, y=-0.000m, z=0.100m, vz=0.000m/s
t=1.002s, x=0.000m, y=-0.000m, z=0.100m, vz=0.000m/s
t=1.202s, x=0.000m, y=-0.000m, z=0.100m, vz=0.000m/s
t=1.402s, x=0.000m, y=-0.000m, z=0.100m, vz=0.000m/s
t=1.602s, x=0.000m, y=-0.000m, z=0.100m, vz=0.000m/s
t=1.802s, x=0.000m, y=-0.000m, z=0.100m, vz=0.000m/s
```

### 10.2 现象分析

在仿真开始阶段，方块受到竖直向下的重力作用，`z` 坐标逐渐减小，且 `vz` 为负值，说明方块正在向下运动。

在 `t≈0.402 s` 时，方块中心高度达到 `z≈0.099 m`，同时竖直速度变为很小的正值。这说明方块下表面与地面发生接触后，MuJoCo 的碰撞约束对方块状态进行了向上的修正。

在 `t≈0.602 s` 之后，方块中心高度稳定在 `z≈0.100 m`，且 `vz≈0 m/s`，说明方块已经稳定静止在地面上。

整个过程中，`x` 和 `y` 坐标始终接近 `0`，说明方块没有发生明显横向偏移。输出中出现的 `-0.000 m` 属于浮点数计算和格式化显示导致的微小误差。

### 10.3 曲线观察结论

通过运行 `src/simulate_box_plot.py` 绘制高度随时间变化的曲线，可以观察到：

* 接触地面前，方块高度曲线呈越来越快下降的弯曲趋势；
* 方块接触地面后，曲线存在很小的波动和修正过程；
* 最终方块高度稳定在 `z≈0.100 m`。

相比每隔一段时间打印一次状态，曲线能够更连续地展示方块下落、接触地面和最终稳定的全过程。
