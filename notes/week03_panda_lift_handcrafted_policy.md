# Week 03 - Panda Lift 手工策略与 RL 训练可行性判断

## 1. 实验目标

本实验基于 robosuite Panda Lift 环境，在随机策略 baseline 的基础上，进一步实现分阶段手工策略，用于分析机械臂操作任务中的动作接口、任务阶段、奖励变化和后续强化学习训练可行性。

本阶段不急于直接训练复杂强化学习模型，而是先通过手工策略判断 Panda Lift 任务是否具有清晰的完成路径，以及当前环境配置是否适合继续进行 RL 训练。

## 2. 任务阶段分析

Panda Lift 任务可以拆分为四个阶段：

1. 移动末端执行器到方块上方；
2. 下降到方块抓取位置；
3. 闭合夹爪夹取方块；
4. 保持夹爪闭合并向上抬起方块。

本实验使用有限状态机实现上述过程。

## 3. 手工策略设计

策略主要使用以下 observation：

- `robot0_eef_pos`：机械臂末端执行器位置；
- `cube_pos`：方块位置。

策略输出 action，其中：

- `action[:3]` 控制末端执行器在 x、y、z 方向上的运动；
- `action[-1]` 控制夹爪开合。

手工策略中设置了四个 phase：

- phase 0：移动到方块上方；
- phase 1：下降到抓取位置；
- phase 2：闭合夹爪；
- phase 3：向上抬起方块。

## 4. 实验结果

### 4.1 随机策略 baseline

此前随机策略运行 5 个 episode，结果如下：

- Episode reward: 1.157, 0.371, 13.196, 3.434, 0.594
- Average reward: approximately 3.75
- Success rate: 0.0
- Max steps: 200

随机策略未能完成 Panda Lift 任务，说明该任务不能依赖随机探索完成，需要明确的阶段式操作策略。

### 4.2 手工策略结果

手工策略运行结果中，phase 能够按照预期从 0 切换到 3。实验输出显示，在 phase 3 阶段 reward 达到 1.000，且视频中可以观察到方块被夹爪抓起并抬升。

一次运行结果如下：

- Average total reward: 194.449
- Average success steps: 0.000
- 视频观察结果：方块已被夹爪抓起并抬升

虽然环境返回的 `success_steps` 为 0，但从视频和 reward 变化来看，手工策略已经完成了有效抓取和抬升动作。该现象说明 robosuite 内部的 `success` flag 判断可能与可视化结果存在阈值差异，后续实验不能只依赖 `info["success"]`，还应结合视频、方块高度和 reward 进行综合判断。

## 5. 现象分析

与随机策略相比，手工策略的 average reward 明显提高，说明分阶段策略能够有效引导机械臂完成接近、下降、闭爪和抬升过程。

实验中观察到：

- 末端执行器能够移动到方块附近；
- 夹爪能够在接近方块后闭合；
- 抬升阶段中方块随夹爪上升；
- reward 在后期达到较高水平。

因此，当前环境的动作接口、observation 读取和任务阶段划分是有效的。

## 6. RL 训练可行性判断

根据手工策略实验结果，Panda Lift 任务具备后续 RL 训练的可行性。

原因如下：

1. 任务阶段清晰，可以拆分为接近、下降、抓取和抬升；
2. observation 中包含末端位置和方块位置，能够支持低维状态训练；
3. action 接口可以有效控制末端运动和夹爪开合；
4. 手工策略已经能够在视频中完成抓取和抬升，说明任务不是环境配置错误，而是策略学习问题。

不过，考虑到 Panda Lift 比单关节摆杆任务复杂得多，后续不建议直接进行大规模视觉 RL 训练，而应优先尝试低维 observation 下的 RL 训练，算法可优先考虑 SAC 或 TD3。

## 7. 阶段结论

本实验完成了 Panda Lift 任务的随机策略 baseline 和手工策略 baseline 对比。随机策略无法完成任务，而手工策略可以通过有限状态机完成有效抓取和抬升，说明该任务具有明确的操作结构和后续 RL 训练价值。

从求职展示角度看，本阶段成果体现了对机器人操作任务的分析能力，包括任务分解、动作接口理解、baseline 构建、失败现象分析和 RL 可行性判断。
